# voice_profile の作り方ガイド

## voice_profile とは

あなたの「書き言葉の指紋」を分析した資料。AIがあなたの声で記事を書くための参照データ。

**重要な教訓:** 声は分解しても再現できない。voice_profileは「AIがあなたの声を完全に再現するためのもの」ではなく、「AIが最低限やってはいけないことを知るためのもの」。完全な再現は無理。方向性を合わせることが目的。

---

## 素材の集め方

### ChatGPTのチャット履歴から

1. ChatGPTの設定 → データコントロール → データのエクスポート をリクエスト
2. conversations.json を取得
3. 以下のスクリプトで自分の発言だけを抽出:

```python
import json

with open('conversations.json', 'r') as f:
    data = json.load(f)

messages = []
for conv in data:
    for node_id, node in conv.get('mapping', {}).items():
        msg = node.get('message')
        if msg and msg.get('author', {}).get('role') == 'user':
            parts = msg.get('content', {}).get('parts', [])
            text = ' '.join([p for p in parts if isinstance(p, str)])
            if len(text) > 20:  # 短すぎるものは除外
                messages.append(text)

with open('my_messages.txt', 'w') as f:
    for m in messages:
        f.write(m + '\n---\n')

print(f"抽出完了: {len(messages)}件")
```

### Claudeのチャット履歴から

Claude.aiの会話をエクスポートし、同様に自分の発言だけ抽出。

### その他の素材

- SNSの投稿（X/Twitter、Facebook等）
- ブログ記事
- メールの文面
- 日記・メモ

**量の目安:** 500件以上あると精度が上がる。1,000件以上が理想。

## 分析の手順

抽出したテキストをClaude（またはChatGPT）に渡して分析させる。以下のプロンプトを使用:

```
以下は私のチャット履歴から抽出した発言です。
この発言群を分析して、以下の項目を抽出してください:

1. **文長分布:** 平均文長、最短・最長、分布の特徴
2. **語末パターン:** 文末の言い回しのバリエーション（「〜だ」「〜だな」「〜かも」等）
3. **一人称:** 使用している一人称とその使い分け
4. **口癖・定型句:** 頻出するフレーズや言い回し
5. **思考パターン:** 具体→抽象、比喩の傾向、論理展開の癖
6. **感情表現:** 感情をどう表現するか（直接的/間接的、頻度）
7. **メタファーの傾向:** どの領域からメタファーを持ってくるか
8. **敬語の使い方:** 常体/敬体の切り替えパターン
9. **ユーモアの型:** どういう笑いを使うか（自虐、ツッコミ、皮肉等）

具体的な発言例を引用しながら分析してください。
```

分析結果を以下の構成でまとめる:


## 使い方の注意（重要）

### voice_profileは「方向づけ」であって「設計図」ではない

このパイプラインを運用して得た最大の教訓:

**声は分解しても再現できない。**

voice_profileで「文長SD」「語末パターン」「接続詞の傾向」を定量的に分析しても、それを全部AIに指示しても、出てくる文章は「本人が書いたもの」にはならない。

なぜか:
- 人間の文章の魅力は「予測不可能性」にある。パターンに従うことではなく、パターンを破ることが面白さを生む
- voice_profileは「平均」を記述する。しかし人間の声は「平均からのズレ」にある
- AIにvoice_profileを渡すと、AIは「全てのルールを同時に守ろう」とする。結果、どのルールも中途半端になる

### voice_profileの正しい使い方

1. **writerの参考資料として:** 「こういう方向で書け」という方向づけ。完全な再現は求めない
2. **voice_checkerの比較対象として:** 「この文章は{OWNER}が書いたように聞こえるか」のチェックに使う
3. **calibratorの分析軸として:** AIと本人の乖離を4層で分析する際の基準値

### voice_profileより効果的なもの

- **OK記事**（Calibration/ok/）— 手直し後の完成形が、最も具体的な「声のサンプル」
- **voice-patterns.md** — AIがやりがちなミスの蓄積。「何を避けるか」は「何をするか」より効果的
- **構想の語り** — {OWNER}が構想に書いた生の言葉が、最も本人の声に近い

## 最初はこれだけでOK

完璧なvoice_profileを作ってから始める必要はない。以下だけあれば動く:

1. 一人称のルール
2. 禁止フレーズ（AI臭い表現のリスト）
3. 口癖や特徴的な表現 3-5個

残りはcalibratorが学習サイクルで育てる。3記事も回せば、voice-patterns.md の方がvoice_profileより実用的になる。
