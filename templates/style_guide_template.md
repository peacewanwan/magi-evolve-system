## 3. 禁止表現

AIが生成しがちな表現のうち、あなたの文体に合わないものをリストアップする。

| 禁止フレーズ | 理由 |
|------------|------|
| 「いかがでしたか」 | 上から目線 |
| 「〜ではないでしょうか」 | 曖昧・逃げ |
| 「参考になれば幸いです」 | 無意味な謙虚 |
| 「本記事では〜をご紹介します」 | 説明的すぎ |
| 「〜させていただきます」 | 過剰敬語 |
| 「今回は〜について」 | テンプレート導入 |
| 「以下、詳しく解説します」 | 不要なつなぎ |
| 「〜という観点から」（頻発） | AI臭 |
| 「〜が重要です」（連発） | AI臭 |
| 「〜と言えるでしょう」 | AI臭 |
| 「まず〜、次に〜、最後に〜」 | テンプレート構成 |
| {あなた固有の禁止表現} | {理由} |

## 4. リズムと息継ぎ

**{あなたの密度方針を一言で}**
<!-- 例: 「濃いけど、詰まりすぎない。」 -->

| 項目 | 基準 |
|------|------|
| 全体文字数 | {目安: 2,000〜5,000字} |
| 1段落 | {目安: 最大3-4行} |
| 1文 | {目安: 60〜80字} |
| 見出し（H2） | {目安: 2〜4個} |

### 息継ぎのルール

データや論理が3段落以上続いたら、**息継ぎを1行入れる。** 短い感想、自己ツッコミ、問いかけ。

### 見出しは「語り」にする

見出しは目次ではなく、語りの一部。

```
❌ レポート型見出し
## データの比較
## 結果の考察

✅ 語り型見出し
## で、実際どのくらいズレてたか
## 結局そういうことだった
```

## 5. 記事構成

### タイトル

| 項目 | 基準 |
|------|------|
| 文字数 | **32文字以内** |
| 構成 | 主張か矛盾を含む。説明ではなく断言 |

### 冒頭

- 3行以内で本題に入る
- 「この記事では〜」という宣言は不要

### 終わり方

- 言い切りで終わっていい
- 大きな教訓を引き出そうとしない

## 7. 記事の型（3種類）

### 型1: 構造を暴く
具体的な事象の裏にある構造を見せる。

### 型2: 実測で語る
自分のデータで一般論を上書きする。

### 型3: 考えが変わった
前提が覆った過程を見せる。

## 8. 品質基準

### 必須（これがないと出さない）

- [ ] 一次情報（自分の体験・実測・判断）が記事の核にあるか
- [ ] AI生成っぽい定型表現がないか
- [ ] 一人称が適切か
- [ ] 禁止フレーズがないか
- [ ] 「レポート」ではなく「エッセイ」になっているか

### 推奨

- [ ] タイトルが具体的で、主張か矛盾が含まれているか
- [ ] 冒頭3行で本題に入っているか
- [ ] 息継ぎが入っているか
- [ ] 見出しが「語り」口調になっているか
