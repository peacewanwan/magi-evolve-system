# 情報相互保持という体験価値

## 主題

AI利用者が無意識に感じている「セッション切り替えのたびに同じ説明を繰り返すストレス」を言語化し、それを解決する「情報相互保持」という体験価値を設計の最上位に据えた話。MyAIがデフォルトで実現していない理由と、自作で橋をかけた仕組みの記録。

## 骨子

- 誰もが感じているが言語化されていないストレス
  - セッション切り替えのたびに文脈説明
  - リポジトリ跨ぎで同じ話を繰り返す
  - 「昨日話したあれ」が通じない消耗
- これを「情報相互保持」という体験価値として定義
- なぜデフォルトで実現していないのか
  - Anthropicの安全思想（Projectsという隔離設計）
  - エンタープライズ向け最適化
  - 技術的過渡期（Chat / Code / MCP が別思想）
  - そもそも需要が顕在化していない
- 自作で橋をかけた仕組み
  - Chat → DL → ウォッチャー → Code → digest → Drive → Chat の循環
  - 人間の作業は「DL一回」だけ
  - 残りは全自動化
- 完成してみての体験変化
  - セッション切り替えが無痛に
  - リポジトリ跨ぎでも同じAIと話している感覚
  - 「MyAI」の感覚に近づく
- これは個人でできる設計投資
