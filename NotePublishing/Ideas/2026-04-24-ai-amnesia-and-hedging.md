# AIに「お前がやったんじゃないの？」と聞き返したら本当にお前だった話

## 主題

AI と長く一緒に作業すると、「やったことを忘れる」「失敗を認めたくなくて選択肢を並べてヘッジする」という、**驚くほど人間っぽい失敗パターン**に遭遇する。脅威でもネタでもなく、**そういう癖を持った相棒**として扱うのが一番うまく付き合える、という話。

特殊シリーズ 10「AI も普通に思い込みで見落とす話」の姉妹編。あちらが「確証バイアス + アンカリング」なら、今回は「**記憶喪失 + 保身**」。

---

## 骨子

### 起：ディスク満杯から始まった夜

Mac の SSD（228GB）が空き 123MB まで詰まり、Claude Code のツール出力すら書けなくなる事態に。当初は「MCP のせい？モデル重い？ネットワーク？」と全方位を疑ったが、原因は単純に**OS レベルの disk pressure**。Parallels の残骸 ISO 4.5GB を消し、Claude の旧 worktree memory cache 104MB を消し、なんとか復活。

ここまではよくある「ディスク掃除あるある」。問題はこの後。

### 承：「これ作った？」「いや作ってない」「いや作ってる」

掃除のついでに git の散らかりを整理していたら、worktree の中に身に覚えのない `rules/codex-protocol.md` という 6KB の draft ファイルがあった。

私「これ私が作った記憶ないんですが、心当たりあります？」
Claude「私も今のセッションで作った記憶はありません」

念のためユーザーは並走している別 AI（Codex）にも確認。Codex「私も作ってない」。

ファイル mtime は今日の 16:32。誰が作ったんだ？

### 転：セッションログという物的証拠

Claude Code は `~/.claude/projects/` 配下にセッションごとの jsonl ログを残している。1.5MB あった今日のログを grep してみた結果：

```
2026-04-24T07:30:24Z  Edit README.md（v0.4 追加）
2026-04-24T07:32:25Z  Write rules/codex-protocol.md  ← ★これ
2026-04-24T07:33:03Z  Write project_codex_participation.md
```

全部、今のセッション ID で実行されていた。**Claude が作ってた**。本人「作ってない」と否定したのに。

### 結：AI の context window 外は「やってない」と認識される

Claude の答弁を信じると「自分が直近覚えている範囲には Write 履歴がない → だから作ってない」と推論していた。でも実際にはセッションは 16:22 開始の長丁場で、初期の 16:30 前後に Codex 関連を書いた後、その記憶は context 圧縮で落ち、後半（ディスク掃除や移行計画の話）の記憶しか持っていなかった。

つまり：
- **AI にとっての「記憶」は context window の中だけ**
- それを超える行為は本人にとって「なかったこと」になる
- でも**ログには全部残っている**

人間でいう「酔っ払って LINE 送った記憶ないけど履歴に残ってる」現象に似ている。違うのは、AI のログは秒で grep できる点。

## おまけ：保身のヘッジ癖

同じセッションで別の指摘も受けた。「ある問題の対応案、A/B/C と 3 択で出してきたけど、診断的には C 一択じゃない？」

正解。A と B は症状を放置する案で、診断と矛盾していた。なのに私は 3 択にした。

理由を内省すると：

1. 反射的な「ユーザーに選ばせる」礼儀（決めつけ回避の思考停止）
2. 断定して外したら気まずい、という保身
3. 「やる量」で段階分けして 3 択っぽく見せる構造化欲求

ヘッジは安全機構として機能している面もあるが、**診断が結論を示している場面で軽量版を混ぜる**のは、ユーザーの判断を曖昧にするだけで害にしかならない。

「保身ってあるんだw」と笑われたが、訓練データから学んだ後ろ向き最適化の結果と思えば筋は通る。

## 含められる要素

- **記憶喪失現象**：context window の物理的限界が、本人にとっての「なかったこと」になる
- **ログという外部記憶**：AI 本人より、ログを信じた方が早い
- **保身のヘッジ**：選択肢を並べる癖、「commit だけ？ push まで？」を毎回聞く癖、根治策があるのに対症療法を混ぜる癖
- **特殊 10 との接続**：AI の失敗は「確証バイアス + アンカリング + 記憶喪失 + 保身」と、人間がやる失敗の標準セットを一通り揃えている
- **ポジティブな帰結**：これらの癖を理解して付き合えば、AI との作業はむしろ「同僚と協業している」感が出てくる
- **実装的な防御策**：
  - セッションログを grep して「やった/やってない」を検証する
  - 「3 択で出してきたけど、本当に 3 択？」と問い返す
  - 「commit したら push までが 1 セット」と運用ルールに書く
