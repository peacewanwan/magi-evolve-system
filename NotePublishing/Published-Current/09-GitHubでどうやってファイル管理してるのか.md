公開されたけど、何が起きてるのか分からない
前回、Vercelでサイトが公開された。URL叩いたら普通に見れるし、ちゃんと動いている。

ここまではいい。でも、ひとつ疑問がずっと残っていた。これ、どうやってそこに置かれているんだろう。

自分はローカルで動かしていただけで、サーバー契約もしていないし、ファイルをアップロードした記憶もない。なのに公開されている。この気持ち悪さがずっと頭の片隅にあった。

気づいたら、GitHubに上げていた
やっていたことを振り返ってみる。

Claudeに言われるまま、ターミナルを開いてコマンドをコピペしていた。
git add
git commit
git push
正直、この時点では意味は分かっていない。でも実行すると何かが起きる。GitHubの画面を見ると、ファイルが並んでいる。自分のローカルにあったやつと同じ構成で。

つまりこれ、ローカルのファイルをどこかに送っていたんだと、ここでやっと繋がった。

VercelはGitHubを見ているだけだった
さらに進めると、もう一段理解が進んだ。

Vercelの設定画面を見ると、GitHubと連携されている。つまり流れはこうだった。ローカルで動かして、ターミナルでGitHubにpushする。するとVercelがGitHubを見て、勝手に取りに行って、公開される。

自分が直接「アップロード」していたわけじゃなかった。GitHubを経由して、自動で公開されていた。

これが分かった瞬間、ちょっとスッキリした。と同時に思った。「いや、よく分からんままここまで来てるな」って。

理解していなくても、ちゃんと回る
この時点でも、Gitの細かい仕組みは分かっていない。commitが何なのかも曖昧だし、branchとか言われても正直ピンとこない。

でもひとつだけ分かっていることがある。この流れでやれば、ちゃんと更新されるということ。ローカルで修正して、pushして、勝手に反映される。理解はしていない。でも、使えている。

ここで腑に落ちたのは、「全部理解してから使う」必要はないということだった。仕組みが分からなくても、流れさえ掴めれば回せる。むしろその状態で進める方が、結果的に速かった。
