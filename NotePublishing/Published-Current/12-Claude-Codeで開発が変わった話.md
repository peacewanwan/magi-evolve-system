# コードほぼ書けないのに、Claude Codeで開発が変わった話 〜AIで素人がWebサービス作る話⑫〜

公開日: 未公開（下書き 2026-04-08）
スキ: -

---

これまでのやり方
これまでの開発は、全部同じやり方だった。

Claudeのチャットにやりたいことを伝える。コードが返ってくる。それをコピーして、ターミナルに貼り付けて、実行する。エラーが出たらそのまま貼って聞く。修正されたコードが返ってくる。またコピペする。

この繰り返しで、Vercelへのデプロイも、Supabaseの接続も、クローラーの実装も、全部進めてきた。動くことは動く。でも正直、かなり手間がかかっていた。

「Claude Codeって何？」
ある日、どこかでClaude Codeという単語を見かけた。何なのかよく分からなかったので、そのままClaudeに聞いた。

「Claude Codeって何？」

返ってきた説明によると、ターミナル上で動くコーディングアシスタントらしい。今まではClaudeとブラウザのチャット画面でやり取りしていたけど、Claude Codeはターミナルの中で直接動く。ファイルの読み書きもコマンドの実行もAIが直接やってくれるという。

つまり、これまで自分がやっていた「コードをコピーして、貼り付けて、実行する」という作業がまるごとなくなるということだった。

インストールでいきなり詰まった
さっそくMacに入れようとした。Claudeに教えてもらった通りにインストールコマンドを実行して、ターミナルにclaudeと打ち込んだ。

command not found。

動かない。またか、と思った。Supabaseを繋いだ回と同じで、言われた通りにやっているのに動かないパターンだった。

そのままエラーをClaudeに投げたら、ターミナルがClaude Codeの場所を認識できていないという話だった。設定ファイルに場所を教えてあげる必要があるらしい。言われた通りに編集して、ターミナルを再起動した。

もう一度claudeと打つ。今度はテーマ選択の画面が出てきた。動いた。

コピペがなくなった
Claude Codeが動くようになってから、開発の流れが一変した。

今までは「コードをもらう → コピーする → 貼り付ける → 実行する → エラーを貼る」の5ステップだった。Claude Codeだと「やりたいことを伝える」の1ステップで終わる。ファイルの編集も、コマンドの実行も、エラーの確認も、全部Claude Codeが直接やってくれる。

自分がやるのは、何をしたいかを日本語で伝えることだけになった。

最初に気になったのは「引き継ぎ」だった
Claude Codeが動いた直後、すぐに気になったことがあった。「今までチャットで決めてきた設計とか、どうやってClaude Codeに伝えるの？」ということだった。

ブラウザのClaudeには「プロジェクト」という機能がある。あらかじめ設計資料やルールを登録しておくと、チャットのたびにそれを踏まえて答えてくれるというもの。これのおかげで、毎回ゼロから説明しなくても話が通じるようになっていた。

でもClaude Codeはターミナルで動く別のツールなので、チャット側に登録した情報は見えない。何十回もやり取りして固めてきたDB設計やクローラーの仕様を、また一から説明し直すのかと思うと気が重かった。

聞いてみたら、CLAUDE.mdという名前のファイルを、自分のコードが入っているフォルダの一番上に置いておけば、Claude Codeが毎回それを読んでくれるらしい。GitHubにpushしているあのフォルダのことだった。やっていることはチャット側のプロジェクト機能と同じで、「この開発の前提知識はここに書いてあるから読んでね」というメモを渡す感覚に近かった。

これで、チャットで積み上げてきた知識をClaude Codeに引き継げるようになった。

振り返ると
コピペで開発していた期間は、正直そこまで長くなかった。でもあの時期があったからこそ、Claude Codeのありがたみが分かった。

作り始めの回で「止まることの方が問題」と書いた。Supabaseを繋いだ回で「毎回同じパターンで前に進む」と書いた。Claude Codeの導入もまったく同じだった。よく分からないけど入れてみる。詰まったらエラーを投げる。動いたら使う。理解は後から追いついてくる。

ただ、ひとつだけ違うことがある。これまでは「AIにもらったものを、自分の手で組み立てていた」けど、ここからは「AIが直接組み立てる」に変わった。自分の役割は、作業者から指示を出す側に変わったということだった。
