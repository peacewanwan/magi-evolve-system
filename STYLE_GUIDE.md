# STYLE GUIDE — 文体・ルール

## 基本文体
- **常体（だ・である調）** が基本
- 敬体は意図的なミックスとして使う（「〜してくれる」「〜させてもらっている」程度）
- ですます調で統一するのは禁止

## 一人称
- **「自分」** 一択
- 「僕」「私」「俺」「筆者」は使わない

## 文の長さ
- 一文は短く。迷ったら切る
- 体言止め・名詞止めを積極的に使う
- 「〜だった。〜だと思う。」のように短文を連ねるリズム

## 口語表現（積極的に使う）
- 「〜なんだよね」「〜じゃん」「〜だけどさ」
- 「〜ってのも」「〜ってくらい」
- 「いったん」「ひとまず」「とりあえず」
- 「〜してみるか」「〜かなと思ってる」

## 感情・反応の出し方
- 短く出す。説明しない
- OK: 「これがデカい」「いい感じ」「撃沈した」「知らんかった」
- NG: 「非常に大きな影響があった」「かなり良い結果となった」

## 自虐・ユーモア
- 控えめに自然に入れる
- おじさん感を消さない
- 「分からない」「知らなかった」は素直に書く
- 自虐はあるけど卑屈にはしない

## 構成
- 見出しは3〜5個程度。細かく刻みすぎない
- 時系列ベースで自然に。きれいに整理しすぎない
- 脱線OK。ただし戻ってくる
- 結論や教訓は最後にさらっと。説教にしない

## 禁止表現・パターン
- 「〜ではないでしょうか」— 問いかけ風の押しつけ
- 「〜と言えるでしょう」— 評論家口調
- 「まとめると〜」で始まる教科書的なまとめ段落
- 「いかがでしたか？」系
- 「〜について解説します」— 解説記事のトーン
- 「それでは見ていきましょう」— ガイド口調
- 「〜することが重要です」— 啓蒙口調
- 過剰な接続詞（「しかしながら」「一方で」「加えて」の連発）
- 「筆者は」「私たちは」
- 箇条書きの多用（本文中。設計ドキュメントならOK）

## 絵文字
- 基本は使わない
- 意外性として時々入れるのはあり（多用禁止）

## タイトルルール
- **本編:** `コードほぼ書けないのに、〇〇〇〇 〜AIで素人がWebサービス作る話⑬〜`
- **番外編:** `【番外編◯】〇〇〇〇`
- 「コードほぼ書けないのに、」が必ず冒頭に入る（本編のみ）
- 丸数字で通し番号

## 記事末尾テンプレ
```
ここまで読んでくれてありがとう。

Xもやってるので、よかったら覗いてみてほしい。
個人アカウント：https://x.com/Peacewanwan?s=20
オフマップ公式：https://x.com/24offmap_jp?s=20

サイトはこちら：https://24offmap.jp
```

## 密度
- 量より質。短く濃く
- 水増ししない。書くことがなければ短くて終わり
- 同じことを言い方を変えて繰り返さない
