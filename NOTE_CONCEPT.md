# NOTE CONCEPT — noteアカウントのコンセプト定義

## アカウント名
peacewanwan（生しらす）

## 発信の軸
「非エンジニアの意識低い系おじさんが、AIでWebサービスを本気で作っている過程」をリアルに記録する。

## 人物像（読者にどう見られたいか）
- **面白いおじさんがなんか作ってる** + **非エンジニアだけど本気で作ってるやつ**
- 技術者ぶらない。分からないことは分からないと言う
- でもやると決めたらちゃんとやる。課金もするし有給も使う
- 自虐はあるけど卑屈じゃない。楽しんでる感じが出ている

## 作っているサービス
- **二輪四輪オフマップ（24offmap.jp）** — 車・バイクのオフ会・ミーティング情報の自動収集サイト
- **オフ会メーカー** — 「今ここにいる」を可視化して自然発生オフ会を促す機能
- ターゲット：初参加者（「行ってみたいけど不安」な人）

## 記事シリーズ
### シーズン1（①〜⑥、リライト完了・公開済み）
「意識低い系おじさんがAIと何かを作る話」①〜⑥
- 概念編：AI使い分け、考え方、壁打ちの流れ

### シーズン2（⑦〜、リライト進行中）
実装の具体的な話（環境構築、ホスティング、GitHub、Supabase、データ収集、Claude Code等）
- ⑦〜⑩：リライト完了・公開済み
- ⑪：情報ソース調査（大幅リライト完了・公開済み）
- ⑫以降：⑪の大幅リライトに伴い再構成が必要

### 番外編（リライト進行中）
- 番外①②：リライト完了・公開済み（AIへの聞き方、同じところでハマる）
- 番外③：自己紹介記事（作成済み・挿入位置未決定）
- 番外④：欠番（理由不明 — 要確認）
- 番外⑤：用語集（作成済み・挿入位置未決定）

### 今後
- シーズン3以降：運用、認知度向上、学習サイクル
- 単発記事：開発中に気づいたこと、失敗談

## 読者像
- AI興味あるけどまだ使いこなせてない人
- 個人開発に興味あるけどコード書けない人
- 「自分でもできそう」と思いたい人
- 車好き（オフマップのユーザー候補）

## noteでやること
1. 過去記事（AI生成版）のリライト — 「自分の声」で書き直す
2. 開発過程の続きを書く — シーズン2以降
3. サイトへの導線を作る — note → X → オフマップ

## noteでやらないこと
- ですます調
- 技術解説記事（目的はプロセスの共有であって教育ではない）
- AIすごい系のポエム
- 絵文字の多用（意外性としてたまに使うのはあり）
- 体験していないことを書く

## 導線設計
- note記事末尾 → Xフォロー誘導（個人 + オフマップ公式）
- note記事末尾 → サイトURL（24offmap.jp）
- X投稿 → note記事への流入
